# SDK 0.2.108 → 0.2.141 完全重构

> 日期: 2026-05-14 | 决策: 方案 D（完全重构，无防腐层）
> 状态: 已批准 | 预计工期: 5~6 工作日

## 动机

`@anthropic-ai/claude-agent-sdk` 从 0.2.108 到 0.2.141 跨越 33 个 patch 版本，结构彻底变化：

- 移除 `cli.js`（bun-hosted JS → 原生二进制）
- 移除 `embed.js` / `extractFromBunfs.js` / `vendor/`
- CLI 二进制拆分到 9 个 optionalDependencies 平台包（每个 ~200MB）
- ESM-only（移除 CJS shim）
- 默认 spawn 方式从 `bun cli.js` 变为直接执行原生二进制

**设计原则**：借升级做一次彻底清理——删除所有为旧形态（cli.js + bun + preload interceptor）存在的代码，不引入防腐层抽象。

## 决策上下文

评审了 5 个方案：

- A: 纯升级 + interceptor 降级
- B: 升级 + HTTPS MITM 代理
- C: 不升级
- D: **完全重构**（选中）
- E: 重构 + 防腐层

选 D 的理由：

1. 团队当前在重构周期内（Wave 1 刚完成 SessionManager 拆分）
2. Network Inspector 详情不是核心价值，token 视图可接受
3. 防腐层在当前 SDK 升级频率下（12 月内预计 1 次破坏性变更）ROI 不足
4. 使用第三方 Provider（非直连 Anthropic），跨 Provider 防腐无价值

## 已确认的 SDK 差异

| 项目                         | 0.2.108                     | 0.2.141                                                 |
| ---------------------------- | --------------------------- | ------------------------------------------------------- |
| CLI 入口                     | `cli.js`（bun/node 执行）   | 原生二进制 `claude` / `claude.exe`                      |
| 二进制位置                   | 同包内 `cli.js`             | 平台包内 `claude-agent-sdk-{plat}-{arch}[-musl]/claude` |
| `executable` 字段            | 可用 `"bun"`                | 忽略（无意义）                                          |
| `pathToClaudeCodeExecutable` | 传 `cli.js` 路径            | 传原生二进制路径                                        |
| `--preload` 注入             | 可用（bun flag）            | 不可用（原生二进制）                                    |
| `spawnClaudeCodeProcess`     | 钩子可用                    | 钩子可用，签名不变                                      |
| package `files`              | 含 vendor + embed + extract | 仅 sdk.mjs + d.ts + manifest + bridge/assistant         |
| optionalDeps                 | 无                          | 9 个平台包                                              |

## 影响面

### 删除（~1100 行）

- `features/agent/interceptor/` 整个目录（3 文件，~600 行）
- `features/agent/session/lifecycle.ts::createSpawnOverride`（~80 行）
- `claude-code-utils.ts` 旧函数：`resolveSDKCliPath` / `resolveInterceptorPath` / `resolveClaudeCodeExecutable`（~70 行）
- `scripts/build-interceptor.ts`（~50 行）
- `resources/fetch-interceptor.js`（构建产物）
- Network Inspector UI 详情页部分（~300 行旧 → 200 行新）

### 新增（~840 行）

- `resolveClaudeBinary()` + platform/musl 探测（~40 行）
- `warm-pool.ts`（~80 行，可选 Wave 4a）
- RequestTracker SDK 消息适配（~120 行）
- Token Usage UI panel（~200 行）
- 新测试（~400 行）

## Wave 分解

### Wave 0: 依赖 + 构建（0.5d）

- `@anthropic-ai/claude-agent-sdk: 0.2.141`
- electron-builder: `asarUnpack` 平台包；移除 interceptor extra；评估移除 bun
- macOS entitlements: `com.apple.security.cs.disable-library-validation`
- 删除 `scripts/build-interceptor.ts` 与 `resources/fetch-interceptor.js`

### Wave 1: claude-code-utils 重写（0.5d）

- `resolveClaudeBinary(override?)` — 单一入口
- `pickPlatformTag()` — platform/arch + linux glibc/musl 探测
- 删除旧三套路径解析；保留 `detectRtkHookInSettings`

### Wave 2: session 初始化重构（1d）

- 删除 `createSpawnOverride`
- `pathToClaudeCodeExecutable = resolveClaudeBinary(custom)`
- 删除 `executable: "bun"`、networkInspector 分支、5s watchdog
- `buildSessionEnv` 简化（bunDir/rtkDir 不拼 PATH）

### Wave 3: RequestTracker + 网络 UI（1.5d）

- 删除 `interceptor/` 目录
- RequestTracker 改为消费 `SDKMessage.usage`
- `RequestSummary` 裁剪：保留 token 字段，删除 URL/headers/body
- Network Inspector UI → Token Usage 视图

### Wave 4a: warm-pool（0.5d，可选）

- 新增 `warm-pool.ts`: 预热 1 个 `WarmQuery`
- `SessionManager.createSession` 优先消费 warm

### Wave 4b: plugins 迁移评估（0.5d，可选）

- 评估 SDK `plugins: SdkPluginConfig[]` 能否替代 `mergeAgentContributions`

### Wave 5: 测试 + 文档（1d）

- ACP snapshot 重录；新增 warm-pool / resolve-binary 测试
- 更新 main/Renderer CLAUDE.md 文件清单
- 标记旧 design doc superseded；本文档即 ADR

## 风险

| 风险                    | 缓解                                          |
| ----------------------- | --------------------------------------------- |
| macOS 公证失败          | Wave 0 day 1 跑通，失败即停                   |
| Linux musl 探测不可靠   | Alpine Docker 实测；fallback `/lib/ld-musl-*` |
| optionalDeps 不装       | CI 加 `--include=optional`                    |
| 老 session 数据残留     | 持久化加 schema 版本                          |
| 用户依赖 inspector 详情 | 公告 + UI 提示                                |

## 验收标准

- `bun dev` 能创建会话、收消息流
- `bun build:mac` 出包 + notarize 通过
- 全量测试绿（含重录 snapshot）
- Network Inspector 显示 token usage（无 URL/body）
- 首次会话延迟 warm-pool 命中时 < 1s
